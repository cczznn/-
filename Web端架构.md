# Web端架构

## 一、Web 端架构设计核心思路

以 “**大屏场景适配 + 移动端协同**” 为核心，聚焦大学生在 Web 端的 4 类高频场景，既延续移动端 “轻操作、快响应” 优势，又依托 Web 端大屏特性强化 “深度处理、高效管理” 能力，同时保持与移动端一致的 “普通学习者、辅导者” 双角色区分。

1. 深度资源处理（如电脑端编辑笔记、整理真题文档、批量上传课程资料）；
2. 多维度协作管理（如小组任务看板、辅导者批量处理预约、在线协同备课）；
3. 数据化学习分析（如查看学习时长趋势图表、导出学习报告、分析错题分布）；
4. 跨端场景衔接（如 Web 端收藏资源同步至移动端、移动端发起的辅导预约在 Web 端跟进）。

## 二、Web 端详细功能架构（分层拆解）

Web 端架构分为 “**核心业务层（全角色通用）、角色专属层（差异化功能）、基础支撑层（技术保障）** ”，覆盖从用户登录到完成深度操作的全流程，同时实现与移动端数据实时同步。

### （一）核心业务层：全角色通用功能（Web 端高频入口）

核心业务层是 Web 端所有用户的基础操作入口，聚焦 “大屏信息高效呈现、复杂操作简化流程”，共 5 大模块，每个模块拆解至 “子页面 + 核心功能 + 交互细节”，并与后端微服务（用户服务、资源服务等）直接对接。

### （三）基础支撑层：保障 Web 端稳定运行与体验

基础支撑层是 Web 端功能落地的技术保障，覆盖 “性能、安全、兼容性、跨端同步”，确保大学生在不同浏览器、不同电脑配置下都能流畅使用，同时实现与移动端、后端的无缝对接。

| 支撑类型     | 技术细节                                                     | 落地效果                                                     |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 前端技术框架 | - 核心框架：采用 Vue3+Element Plus，兼顾组件复用性与大屏适配性；- 状态管理：用 Pinia 管理全局状态（如用户信息、登录状态），确保跨页面数据同步；- 路由管理：用 Vue Router 实现路由懒加载，减少首屏加载时间。 | - 首屏加载时间≤2.5 秒（普通网络环境）；- 支持 “响应式布局”（适配 13 寸 - 27 寸屏幕，自动调整组件位置和大小）；- 页面切换无卡顿，操作流畅度≥95%。 |
